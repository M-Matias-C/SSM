ENDPOINTS DISPONÍVEIS
================================================================================

POST   /api/v1/tracking                        → Criar rastreamento
GET    /api/v1/tracking/:id                    → Ver status
POST   /api/v1/tracking/:id/etapa             → Adicionar etapa
GET    /api/v1/tracking/:id/qr                → Gerar QR code
POST   /api/v1/tracking/:id/verify            → Verificar autenticidade
POST   /api/v1/tracking/:id/cancelar          → Cancelar
GET    /api/v1/tracking/meus/rastreamentos    → Meus rastreamentos
GET    /api/v1/tracking/farmacia/rastreamentos → Farmácia
GET    /api/v1/medicamento/:id/history        → Histórico
GET    /api/v1/tracking                       → Estatísticas

COMO USAR
================================================================================

1. Validar Implementação:
   $ cd saude-na-mao
   $ node validate-tracking.js
   $ node quick-test.js

2. Iniciar Backend:
   $ cd backend
   $ npm run dev

3. Iniciar Frontend:
   $ cd frontend
   $ npm run dev

4. Acessar:
   - Rastreamento: http://localhost:3000/rastreamento/:id
   - Verificação: http://localhost:3000/verificar-medicamento

5. Testes:
   $ cd backend
   $ npm test -- src/tests/tracking.test.js

ESTRUTURA DE DADOS
================================================================================

MedicineTracking:
  _id: ObjectId
  medicamento_id: ObjectId (ref: Product)
  lote: String
  farmacia_origem: ObjectId (ref: Pharmacy)
  cliente_destino: ObjectId (ref: User)
  blockchain_hash: String (SHA-256)
  qr_code_data: String (JSON)
  status: 'pendente|em_transito|entregue|cancelado'
  autenticidade_verificada: Boolean
  farmaceutico_validador: ObjectId
  assinatura_farmaceutico: String
  
  etapas: [
    {
      tipo: 'SAIDA_FARMACIA|EM_TRANSITO|ENTREGA|ENTREGUE'
      timestamp: Date
      localizacao: { lat, lng }
      assinatura_blockchain: String
      validado_por: ObjectId
      foto_prova: String
      observacoes: String
    }
  ]
  
  criado_em: Date
  atualizado_em: Date
